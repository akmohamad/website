What is React?

In this article we will learn what ReactJs is and why we should use it instead of other JavaScript frameworks like Angular. What is React JS? React (reactjs) is basically an open-source JavaScript library for creating user interfaces for single page applications. This library is used to manage the View layer for the web. React also has this possibility. It allows us to create reusable UI components. React was originally created by Jordan Walke, one of Facebook's senior engineers. React was first used in Facebook in 2011 and then in 2012 used on Instagram. React allows developers to create very large web applications that can change the date without reloading the page. The most important goals of React can be considered simplicity, speed and scalability. The main focus of React is on It is on the user interface and only corresponds to the View layer in the MVC architecture. This library can be combined and used with other JavaScript libraries and frameworks such as Angular.

What are the features of React

Let's take a look at the important features of React: JSX In React, instead of using regular JavaScript, JSX is used for templating. jsx is a JavaScript format for creating HTML DOMs in the form of a component. React Native It is a JavaScript framework for developing native mobile applications for two operating systems, Android and iOS, which was introduced in 2015. This framework is based on the JavaScript language and the React library. That is, by mastering React, you can learn React Native. Be far ahead of others. Of course, note that there are fundamental differences between this library and the framework, and to understand these differences, I suggest you read the article on the main differences between React and React Native. Single-Way data flow In React, a set of immutable values ​​between components are passed to HTML tags as properties. Components cannot directly change any properties, but they can pass them to the call back function and help them make the changes. This process is completely with "properties flow down; actions flow up" is known. Virtual Document Object Model React creates an in-memory cache structure. In this structure, if a change has occurred, it updates the DOM. This feature enables the programmer, while changes are made in a wtpi, only the component that has changed will be rerendered. Virtual DOM, like the DOM, is a tree of nodes. which includes elements and their attributes and content as objects. The renderer() method creates a tree of React components and changes in each component cause this node to be updated.
